16 Attribute location : <user|test|meta|usertest> = user #:...
17 Many constructs have an associated class, and each user-defined
18 class has an associated test-class and meta-class. This feature
19 attribute allows one to indicate which class (user, test or meta)
20 the construct in question is associated with. The default is
21 the user class.
22
23 Note that the 'usertest' value is not applicable to all sub-constructs. A
24 different mechanism for specifying multiple classes is being implemented.
25
26 TODO(wmh): Make this a multi-value attribute and remove 'usertest'.
27 config:...
28 FeatureValue user #:...
29 Applies to user-provided class (the most common situation).
30 FeatureValue test #:...
31 Applies to the testclass of the class this construct belongs to. If that
32 class has location 'test' it is an error to specify location value
33 'test' (test classs do not have test classes).
34 FeatureValue meta #:...
35 Applies to the metaclass of the class this construct belongs to. If that
36 class has location 'meta' it is an error to specify location value
37 'meta' (meta classs do not have meta classes).
38 FeatureValue usertest #:...
39 This is only legal in a small number of subconstructs. For example,
40 an 'assoc' may be needed in both user and test class.
41 TODO(wmh): Remove this option from here and add it in 'assoc'.
42 end Attribute location;

402 Attribute select : <ro|rw|rwx|raw> = ro #:...
403 Which accessors are public (the others are protected).
404
405 TODO(wmh): This mechanism is somewhat limited. If we could specify,
406 for each accessor, whether or not it is to be generated, and its
407 visibility, that would be best.
408
409 Suppose we use
410 r = getter
411 w = setter
412 x = reffer
413 i = adj
414 @ = the field itself
415
416 We could specify a particular accessor with:
417 +r = getter is public
418 -r = getter is private
419 #r = getter is protected
420 ~r = getter is package
421 !r = getter is not generated
422
423 And the complete set could be:
424 +r+w
425 +r-w-x
426 etc.
427
428 However, the above would require a different mechanism for defining
429 feature values. Two possibilities:
430 1. the set of feature values is
431 r +r -r #r ~r !r
432 w +w -w #w ~w !w
433 x +x -x #x ~x !x
434 i +i -i #i ~i !i
435 and this feature key allows multiple values separated by ''.
436 (note that 'r' is shorthand for '+r'). We may also want to
437 add support for abbrevs for the complete value (e.g. 'raw'
438 is an alias for '!r!w!x!i').
439 2. we add support for regexp-based feature values:
440 ([@+#~!-]?[rwxi])+
441 (we could add special abbrevs to this regexp ... 'raw' means
442 '!r!w!x!i+@' ... but mapping those abbrevs to their actual
443 semantics would need to be hardcoded into the implementation
444 rather than being specified in the schema file unless we provide
445 remapping syntax).
446 config:...
447 FeatureValue ro #:...
448 The getter is public, setter is protected, no other accessors.
449 FeatureValue rw #:...
450 The getter and setter are public, no other accessors.
451 FeatureValue rwx #:...
452 The getter, setter and reffer are public, no other accessors.
453 FeatureValue raw #:...
454 No accessors are generated.
455 end Attribute select;

724 Attribute kind : <instance|static|cls|closure|initializer|finalizer> = instance #:...
725 Identifies one of a variety of mutually exclusive kinds of methods.
726
727 TODO(wmh): The fact that executables have a feature attribute named kind',
728 combined with us using 'kind to refer to the primary key, is destined to
729 pose confusion. Consider renaming one or the other use.
730 config:...
731 FeatureValue instance #:...
732 A ${kind} defined on a class having a receiver object typed of that
733 class.
734 FeatureValue static aliases #:...
735 A method that does not contain a receiver (e.g. a static method when
736 defined at class scope is a function). However, a function can appear
737 in more places ... at namespace level, within methods, etc.
738 TODO(wmh): Is 'static' just an alias for 'function'?
739 FeatureValue cls #:...
740 This is a special case for handling an issue in Python between metaclasses
741 and @classmethods. It indicates that the method should have a @classmethod
742 decorator. It would be preferable to define such methods as having
743 location 'meta', but in Meta that puts the method in the metaclass instead
744 of being an @classmethod method. Python does not treat @classmethods
745 entirely as methods on the metaclass, which breaks things, hence the need
746 for this special case. This value should be considered an implementation
747 detail and NOT used in user-level code, as it does not have a correlate in
748 most other baselangs. It is used internally by Meta when @classmethods
749 are needed to implement some higher-level meta concept.
750 FeatureValue closure #:...
751 A function that has access to the variables in the scope within which
752 it is lexically defined.
753 FeatureValue initializer aliases #:...
754 Called a constructor in most languages ... the method that initializes
755 state of an instance.
756 FeatureValue finalizer #:...
757 Called a destructor in some languages, cleans up state before an object
758 is freed up (either explicitly or implicitly).
